**确认以下迁移已执行：**

- 在 Supabase SQL Editor 中执行了 `migration_v2_complex_orders.sql`
- 检查新表是否已创建（order_items, game_accounts, account_bindings, order_game_accounts, third_party_rentals, shipping_fees）
- 检查 orders 表是否有新字段（total_deposit, total_shipping_cost, order_number）

**验证 SQL：**

```sql
-- 检查新表
SELECT table_name 
FROM information_schema.tables 
WHERE table_schema = 'public' 
AND table_name IN ('order_items', 'game_accounts', 'account_bindings', 'order_game_accounts', 'third_party_rentals', 'shipping_fees');

-- 检查 orders 表新字段
SELECT column_name 
FROM information_schema.columns 
WHERE table_name = 'orders' 
AND column_name IN ('total_deposit', 'total_shipping_cost', 'order_number');
```

### 3. 数据录入建议

**录入历史数据时的操作流程：**

1. **已出售的设备**
  - 在资产管理中，将设备状态设置为 `retired`
  - 在订单管理中，创建订单时可以选择这些 `retired` 设备
  - 创建历史订单（设置过去的日期）
2. **设备购买记录**
  - 在资产管理中手动添加设备信息
  - 在交易记录中记录购买费用（需要先创建交易记录表单功能）
3. **维护费用**
  - 在交易记录中记录维护费用（需要先创建交易记录表单功能）

### 5. 如果遇到问题

**常见问题排查：**

1. **页面显示错误**
  - 检查浏览器控制台错误信息
  - 检查服务器终端错误日志
  - 确认数据库迁移是否完整
2. **创建订单失败**
  - 确认数据库表结构正确
  - 检查必填字段是否都已填写
  - 查看 API 错误信息
3. **数据不显示**
  - 确认数据已正确保存到数据库
  - 检查 API 查询是否正确
  - 刷新页面
