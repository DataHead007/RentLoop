# RentLoop V2 数据库迁移说明

## 概述

V2 版本引入了对复杂订单的支持，包括：
- 多设备/配件订单（一个订单可包含多个设备）
- 数字版游戏账号管理
- 第三方游戏租赁（从淘宝等平台租游戏）
- 物流费用管理
- 账号设备绑定历史

## 迁移步骤

### 1. 备份数据库

**重要：在执行迁移之前，请先备份数据库！**

### 2. 执行迁移 SQL

#### 步骤 2.1: 创建新表和修改现有表

在 Supabase SQL Editor 中执行：
```sql
-- 执行文件：migration_v2_complex_orders.sql
```

这个文件会：
- 创建所有新表（order_items, game_accounts, account_bindings, order_game_accounts, third_party_rentals, shipping_fees）
- 为 orders 表添加新字段（order_number, total_shipping_cost）
- 创建必要的索引和触发器

#### 步骤 2.2: 迁移现有订单数据

在 Supabase SQL Editor 中执行：
```sql
-- 执行文件：migration_v2_data.sql
```

这个文件会：
- 将现有订单的 item_id 数据迁移到 order_items 表
- 为现有订单生成订单编号

#### 步骤 2.3: 验证数据迁移

执行以下查询验证数据迁移是否成功：

```sql
-- 检查订单项数据
SELECT COUNT(*) as order_items_count FROM order_items;
SELECT COUNT(*) as orders_count FROM orders;

-- 检查是否有订单丢失数据
SELECT o.id, o.customer_name, COUNT(oi.id) as item_count
FROM orders o
LEFT JOIN order_items oi ON o.id = oi.order_id
GROUP BY o.id, o.customer_name
HAVING COUNT(oi.id) = 0;
```

如果最后一个查询有结果，说明有订单没有成功迁移订单项，需要手动处理。

#### 步骤 2.4: 删除旧字段（可选）

**只有在确认所有数据迁移成功且系统运行正常后，才执行此步骤！**

```sql
-- 删除 orders 表的 item_id 列（不再需要）
ALTER TABLE orders DROP COLUMN IF EXISTS item_id;
```

## 新表结构说明

### order_items (订单项表)
- 支持一个订单包含多个设备/配件
- 每个订单项有独立的日租金、押金、数量

### game_accounts (数字版游戏账号表)
- 管理自有数字版游戏账号
- 记录账号类型（主认证/非认证）
- 跟踪账号当前登录的设备

### account_bindings (账号设备绑定历史表)
- 记录账号登录到设备的完整历史
- 支持查询账号在某个时间段的绑定情况

### order_game_accounts (订单账号关联表)
- 记录订单中分配的数字账号

### third_party_rentals (第三方租赁表)
- 记录从淘宝等平台租游戏的完整信息
- 支持延期、押金退回等记录

### shipping_fees (物流费用表)
- 区分发货、退货、自提
- 记录物流公司和快递单号

## 影响范围

### 需要更新的代码

1. **API 路由**
   - `/api/orders` - 需要支持创建/更新多设备订单
   - 新增 `/api/game-accounts` - 数字账号管理
   - 新增 `/api/third-party-rentals` - 第三方租赁管理
   - 新增 `/api/shipping-fees` - 物流费用管理

2. **查询函数**
   - `lib/supabase/queries.ts` - 更新订单查询逻辑

3. **前端组件**
   - `components/orders/OrderForm.tsx` - 支持多设备选择
   - `components/orders/OrderList.tsx` - 显示订单项
   - 新增数字账号管理组件
   - 新增第三方租赁管理组件

## 回滚方案

如果迁移出现问题，可以通过以下方式回滚：

```sql
-- 1. 恢复 orders 表的数据（如果有备份）
-- 2. 删除新表
DROP TABLE IF EXISTS shipping_fees;
DROP TABLE IF EXISTS third_party_rentals;
DROP TABLE IF EXISTS order_game_accounts;
DROP TABLE IF EXISTS account_bindings;
DROP TABLE IF EXISTS game_accounts;
DROP TABLE IF EXISTS order_items;

-- 3. 删除 orders 表的新字段
ALTER TABLE orders DROP COLUMN IF EXISTS order_number;
ALTER TABLE orders DROP COLUMN IF EXISTS total_shipping_cost;

-- 4. 删除序列
DROP SEQUENCE IF EXISTS order_seq;
```

## 注意事项

1. **数据一致性**：确保所有现有订单都成功迁移到 order_items 表
2. **前端兼容性**：迁移后，前端代码需要更新才能正常工作
3. **测试**：在生产环境部署前，请在测试环境完整测试
4. **备份**：再次强调，务必先备份数据库！
