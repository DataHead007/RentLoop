# 混合方案实现说明

## 已完成功能

### 1. 数据库迁移
- 添加了 `auto_created` 字段到 `transactions` 表
- SQL 文件：`supabase/add_auto_created_to_transactions.sql`

**执行方式：**
在 Supabase SQL Editor 中执行：
```sql
ALTER TABLE transactions ADD COLUMN IF NOT EXISTS auto_created BOOLEAN DEFAULT FALSE;
```

### 2. 自动创建交易记录
- 当订单状态变为"已完成"（`completed`）时，自动创建租金收入交易记录
- 自动创建的记录包含：
  - `order_id`: 关联的订单ID
  - `type`: 'income'
  - `amount`: 订单的 `total_amount`
  - `category`: '租金收入'
  - `description`: 自动生成的描述（包含订单编号）
  - `transaction_date`: 订单的 `end_date`
  - `auto_created`: `true`

**实现位置：** `app/api/orders/[id]/route.ts` 的 `PATCH` 方法

### 3. 手动创建交易记录
- 创建了交易记录创建/编辑表单组件：`components/transactions/TransactionForm.tsx`
- 创建了新建交易页面：`app/transactions/new/page.tsx`
- 创建了编辑交易页面：`app/transactions/[id]/edit/page.tsx`
- 支持手动创建收入/支出交易记录
- 支持关联到订单或独立创建
- 支持编辑已存在的交易记录

### 4. 交易记录列表增强
- 在交易列表中添加了"自动"标记（`auto_created` 为 `true` 时显示）
- 添加了编辑按钮
- 改进了"来源"列，显示订单链接或"手动创建"
- 操作列包含编辑和删除按钮

### 5. API 路由
- 添加了交易记录编辑API：`app/api/transactions/[id]/route.ts`
  - GET: 获取单个交易记录
  - PATCH: 更新交易记录
  - DELETE: 删除交易记录

### 6. 查询函数
- 添加了 `updateTransaction` 函数：更新交易记录
- 添加了 `getTransaction` 函数：获取单个交易记录

## 使用说明

### 自动创建交易记录
1. 订单状态从其他状态变为"已完成"时，系统会自动创建租金收入交易记录
2. 如果订单的 `total_amount` 为 0 或已存在该订单的收入交易记录，则不会自动创建
3. 自动创建的记录可以手动编辑或删除

### 手动创建交易记录
1. 进入"交易记录"页面
2. 点击"新增交易"按钮
3. 填写交易信息：
   - 类型：收入或支出
   - 金额：必填
   - 类别：可选（从下拉列表选择）
   - 交易日期：必填
   - 描述：可选
   - 如果是从订单页面创建，会自动关联订单ID
4. 点击"创建交易记录"保存

### 编辑交易记录
1. 在交易记录列表中，点击编辑图标
2. 修改交易信息
3. 点击"保存修改"

### 删除交易记录
1. 在交易记录列表中，点击删除图标
2. 确认删除操作

## 注意事项

1. **数据库迁移**：首次使用前，必须在 Supabase 中执行 SQL 迁移脚本
2. **自动创建逻辑**：只有在订单状态变为"已完成"时才会自动创建，其他状态变化不会触发
3. **重复创建保护**：系统会检查是否已存在该订单的收入交易记录，避免重复创建
4. **手动编辑**：自动创建的记录可以手动编辑，编辑后 `auto_created` 字段保持为 `true`
