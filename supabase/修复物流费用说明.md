# 修复物流费用数据说明

## 问题描述

订单详情页面显示的物流费用（`total_shipping_cost`）与物流费用表中实际记录的总和不一致。

例如：
- 订单表中的 `total_shipping_cost` = ¥10.60
- 物流费用表中的实际总和 = ¥5.30

## 原因分析

可能的原因：
1. 数据迁移时计算错误
2. 手动修改了 `orders` 表的数据
3. 创建订单时计算逻辑的bug（现已修复）

## 修复步骤

### 步骤1：检查数据不一致的订单

在 Supabase SQL Editor 中执行以下查询，查看哪些订单存在数据不一致：

```sql
SELECT 
    o.id,
    o.order_number,
    o.customer_name,
    o.total_shipping_cost AS orders表中的物流费用,
    COALESCE(SUM(sf.amount), 0) AS shipping_fees表中的实际总和,
    o.total_shipping_cost - COALESCE(SUM(sf.amount), 0) AS 差值,
    CASE 
        WHEN o.total_shipping_cost != COALESCE(SUM(sf.amount), 0) THEN '不一致'
        ELSE '一致'
    END AS 状态
FROM orders o
LEFT JOIN shipping_fees sf ON sf.order_id = o.id
GROUP BY o.id, o.order_number, o.customer_name, o.total_shipping_cost
HAVING o.total_shipping_cost != COALESCE(SUM(sf.amount), 0)
ORDER BY o.created_at DESC;
```

### 步骤2：备份数据（可选但推荐）

如果需要，可以先备份相关数据：

```sql
-- 创建备份表（如果不存在）
CREATE TABLE IF NOT EXISTS orders_backup AS SELECT * FROM orders;
CREATE TABLE IF NOT EXISTS shipping_fees_backup AS SELECT * FROM shipping_fees;
```

### 步骤3：执行修复

执行修复脚本，重新计算所有订单的 `total_shipping_cost`：

```sql
UPDATE orders o
SET total_shipping_cost = COALESCE(
    (SELECT SUM(amount) FROM shipping_fees WHERE order_id = o.id),
    0
)
WHERE EXISTS (
    SELECT 1 
    FROM shipping_fees sf 
    WHERE sf.order_id = o.id
) OR o.total_shipping_cost != COALESCE(
    (SELECT SUM(amount) FROM shipping_fees WHERE order_id = o.id),
    0
);
```

### 步骤4：验证修复结果

执行验证查询，确认所有订单的物流费用都已一致（应该返回0行）：

```sql
SELECT 
    o.id,
    o.order_number,
    o.customer_name,
    o.total_shipping_cost AS orders表中的物流费用,
    COALESCE(SUM(sf.amount), 0) AS shipping_fees表中的实际总和,
    o.total_shipping_cost - COALESCE(SUM(sf.amount), 0) AS 差值
FROM orders o
LEFT JOIN shipping_fees sf ON sf.order_id = o.id
GROUP BY o.id, o.order_number, o.customer_name, o.total_shipping_cost
HAVING o.total_shipping_cost != COALESCE(SUM(sf.amount), 0);
```

## 注意事项

1. **执行修复前请先检查**：使用步骤1的查询确认哪些订单需要修复
2. **备份数据**：如果数据很重要，建议先备份
3. **验证结果**：修复后务必执行步骤4验证，确保数据一致性
4. **影响范围**：这个修复只影响 `orders.total_shipping_cost` 字段，不会影响 `shipping_fees` 表中的数据

## 完整的SQL脚本

所有步骤已整合到 `supabase/检查和修复物流费用数据.sql` 文件中，可以按步骤执行。
