# 卡口功能说明

## 功能概述

为镜头类设备添加了卡口类型字段，方便管理不同卡口系统的镜头设备。

## 数据库变更

### 1. 执行 SQL 迁移

需要在 Supabase 数据库中执行以下 SQL 语句：

```sql
-- 文件位置：supabase/add_mount_field.sql
ALTER TABLE items 
ADD COLUMN IF NOT EXISTS mount TEXT;

COMMENT ON COLUMN items.mount IS '卡口类型，主要用于镜头类设备（如 Canon EF、Nikon F、Sony E 等）';
```

**执行步骤：**
1. 登录 Supabase Dashboard
2. 进入 SQL Editor
3. 执行上述 SQL 语句

## 功能特点

1. **智能显示**：只有当选择的品类名称包含"镜头"或"镜"字时，才会显示卡口选择字段
2. **预设卡口列表**：包含常用卡口类型，包括：
   - Canon EF / EF-S / RF
   - Nikon F / Z
   - Sony E / FE / A
   - Fujifilm X / GFX
   - Panasonic L
   - Olympus M.Zuiko
   - Leica M / L
   - Pentax K
   - Sigma SA
   - Micro Four Thirds
   - 其他（自定义输入）

3. **灵活输入**：
   - 可以从预设列表中选择
   - 可以选择"其他"后自定义输入
   - 如果输入的值不在预设列表中，也会显示输入框

## 使用方法

1. **新增镜头设备时**：
   - 选择品类（名称包含"镜头"的品类）
   - 在"设备基本信息"卡片中，会自动显示"卡口类型"字段
   - 可以从下拉列表中选择常用卡口，或选择"其他"后自定义输入

2. **查看设备详情时**：
   - 如果设备有卡口信息，会在"基本信息"卡片中显示

## 注意事项

- 卡口字段是可选的，不填写也可以保存设备
- 只有镜头类设备才会显示卡口字段
- 品类名称判断逻辑：包含"镜头"或"镜"字的品类会被识别为镜头品类
