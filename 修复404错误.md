# 修复 404 错误 - Next.js 开发服务器问题

## 问题诊断

浏览器控制台显示：
- `app-pages-internals.js` - 404 Not Found
- `main-app.js` - 404 Not Found

这些是 Next.js 的内部文件，出现 404 错误说明开发服务器可能没有正常运行。

## 解决方法

### 步骤 1: 停止当前的开发服务器

1. 找到运行 `npm run dev` 的终端窗口
2. 按 `Ctrl + C` 停止服务器

### 步骤 2: 清理 Next.js 缓存

在项目目录下运行：

```bash
cd /Users/zk.he/.cursor-tutor/projects/rentloop
rm -rf .next
```

### 步骤 3: 重新启动开发服务器

```bash
npm run dev
```

### 步骤 4: 等待服务器完全启动

等待终端显示类似以下信息：
```
  ▲ Next.js 14.x.x
  - Local:        http://localhost:3000
  - Ready in X.Xs
```

### 步骤 5: 刷新浏览器

在浏览器中按 `F5` 或 `Cmd+R` 刷新页面

## 如果问题仍然存在

### 检查端口占用

如果端口 3000 被占用，可以：

1. **使用其他端口**：
   ```bash
   npm run dev -- -p 3001
   ```
   然后访问 `http://localhost:3001`

2. **或者停止占用端口的进程**

### 检查 Node.js 版本

确保使用的是 Node.js 18 或更高版本：
```bash
node --version
```

### 重新安装依赖

如果问题持续存在，可以尝试重新安装依赖：
```bash
rm -rf node_modules
npm install
npm run dev
```

## 常见原因

1. **开发服务器启动失败** - 检查终端错误信息
2. **端口被占用** - 使用其他端口或停止占用进程
3. **Next.js 缓存损坏** - 删除 `.next` 目录
4. **依赖问题** - 重新安装 node_modules
