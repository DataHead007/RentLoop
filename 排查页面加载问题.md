# 排查页面加载问题

如果页面一直显示"加载中..."，请按以下步骤排查：

## 1. 检查浏览器控制台

1. 按 `F12` 或 `Cmd+Option+I` (Mac) 打开开发者工具
2. 切换到 **"Console"** (控制台) 标签
3. 查看是否有红色的错误信息

常见错误：
- `Failed to fetch` - API 请求失败
- `NetworkError` - 网络连接问题
- `500 Internal Server Error` - 服务器错误

## 2. 检查网络请求

1. 在开发者工具中切换到 **"Network"** (网络) 标签
2. 刷新页面 (F5)
3. 查看 `/api/items` 请求：
   - 状态码是什么？(应该是 200)
   - 响应内容是什么？

## 3. 检查服务器日志

查看运行 `npm run dev` 的终端窗口，查看是否有错误信息。

## 4. 常见问题和解决方法

### 问题 1: Supabase 连接失败
**症状**: 控制台显示连接错误
**解决**: 检查 `.env.local` 文件中的 Supabase 配置是否正确

### 问题 2: 数据库表不存在
**症状**: API 返回 500 错误，日志显示表不存在
**解决**: 在 Supabase SQL Editor 中执行 `supabase/schema.sql`

### 问题 3: 字段不存在（mount 字段）
**症状**: 错误信息提到 `mount` 字段
**解决**: 在 Supabase SQL Editor 中执行 `supabase/add_mount_field.sql`

### 问题 4: 开发服务器未运行
**症状**: 页面完全无法加载
**解决**: 运行 `npm run dev` 启动开发服务器

## 5. 快速检查清单

- [ ] 开发服务器是否正在运行？ (`npm run dev`)
- [ ] `.env.local` 文件是否存在且配置正确？
- [ ] Supabase 数据库表是否已创建？
- [ ] 浏览器控制台是否有错误？
- [ ] 网络请求是否成功？

## 6. 如果仍然无法解决

请提供以下信息：
1. 浏览器控制台的错误信息（截图或复制文本）
2. 网络请求的状态码和响应内容
3. 服务器终端的错误日志
