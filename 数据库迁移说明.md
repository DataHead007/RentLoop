# 数据库迁移说明

## 问题：创建资产时出现错误

如果遇到 "Failed to create item" 错误，很可能是因为数据库还没有执行迁移，缺少 `mount` 字段。

## 解决方法

### 步骤 1：登录 Supabase Dashboard

1. 访问 https://supabase.com/dashboard
2. 登录你的账户
3. 选择你的 RentLoop 项目

### 步骤 2：执行 SQL 迁移

1. 在左侧菜单中点击 **"SQL Editor"**
2. 点击 **"New query"** 创建新查询
3. 复制以下 SQL 语句并粘贴到编辑器中：

```sql
-- 为 items 表添加卡口（mount）字段
ALTER TABLE items 
ADD COLUMN IF NOT EXISTS mount TEXT;

-- 添加注释说明
COMMENT ON COLUMN items.mount IS '卡口类型，主要用于镜头类设备（如 Canon EF、Nikon F、Sony E 等）';
```

4. 点击 **"Run"** 或按 `Ctrl/Cmd + Enter` 执行 SQL
5. 确认看到成功消息

### 步骤 3：验证迁移

执行完成后，可以在 Table Editor 中查看 `items` 表，应该能看到新增的 `mount` 列。

## 文件位置

SQL 迁移文件位于：`supabase/add_mount_field.sql`

## 注意事项

- 这个迁移是安全的，使用了 `IF NOT EXISTS`，即使字段已存在也不会报错
- 迁移后，所有现有数据的 `mount` 字段值将为 `NULL`（这是正常的）
- 只有新添加的镜头设备需要填写卡口信息
